# CertCoach — UI Design System

> منبع واحد حقیقت برای طراحی رابط کاربری. هر مقدار این سند **دقیقاً از کد فعلی پروژه استخراج شده** — چیزی فرضی یا آرزویی نیست. هدف اینه که هر برنامه‌نویسی (یا جلسه‌ی بعدی خود من) بدون حدس زدن، دقیقاً همین مقادیر رو رعایت کنه.
>
> آخرین به‌روزرسانی: بعد از پیاده‌سازی کامل صفحه‌ی Practice Exam جدید (رنگ‌ها، Fortschritt Drawer، گرید سؤالات).

### 1.2 فونت

- خانواده: **Inter Variable** (`@fontsource-variable/inter`) — دقیقاً طبق اسپک اولیه
- اندازه‌ی متن سؤال: `19px` / وزن `700`
- اندازه‌ی گزینه‌ها: پیش‌فرض `text-base` (۱۶px) / وزن عادی، آیکون گزینه `text-xs font-bold`
- دکمه‌ها: `text-sm` (۱۴px) تا `text-[15px]` (دکمه‌ی Fortschritt) / وزن `600`–`700`
- منوی کناری: `text-sm` (۱۴px) / وزن `500`

### 1.3 Radius

## 2. Spacing System

مبنا: **۴px** (نه ۸px دقیق، ولی همه‌ی مقادیر واقعی مضرب ۴ هستن — استاندارد Tailwind)

مقادیر پرکاربرد واقعی تو کد: `gap-2` (۸px)، `gap-3` (۱۲px)، `gap-4` (۱۶px)، `p-4` (۱۶px)، `p-5` (۲۰px)، `p-6`/`p-8` (۲۴/۳۲px برای کارت‌های بزرگ)، `mt-6` بین بلوک‌های اصلی صفحه.

## 4. Component Specification

### 4.1 دکمه‌ی Fortschritt (Progress Trigger)
- ابعاد: `170×44px`
- Radius: `14px`
- پس‌زمینه: شفاف (حالت عادی) → `rgba(124,58,237,.15)` (hover/active)
- Border: `1px solid rgba(255,255,255,.08)` → `#7C3AED` (hover/active)
- آیکون: `BarChart3` (Lucide)، سفید، ۱۷px

### 4.2 Stats Drawer
- عرض: `420px` (دسکتاپ) / `360px` (تبلت) / `100%` (موبایل)
- ورود: از راست، `250ms ease-in-out`
- پس‌زمینه‌ی پشت: `rgba(0,0,0,.45)` + `blur(8px)`
- بسته‌شدن: کلیک بیرون + دکمه‌ی X + کلید Escape
- محتوا: ۴ کارت (دایره‌ی پیشرفت / تفکیک پاسخ / روند / متریک‌های عملکرد) + ۲ دکمه‌ی پایین (Close / View Analytics)

### 4.3 کارت اصلی سؤال
- Radius: `18px`، Padding: `20px` (موبایل) / `32px` (md+)
- پس‌زمینه: `--color-question-panel`

### 4.4 گزینه‌ی پاسخ
- ارتفاع حداقل: `72px`، Radius: `18px`، Border: `1px`
- انتخاب‌شده (قبل از چک): گرادینت بنفش→آبی
- درست/غلط (بعد از چک): پس‌زمینه‌ی سبز/قرمز کم‌رنگ

### 4.5 دکمه‌ی اصلی (CTA)
- گرادینت بنفش→آبی، Radius `16px`، سایه‌ی نرم بنفش

## 5. Responsive Rules

| Breakpoint | عرض | رفتار |
|---|---|---|
| Mobile | `< 640px` | Sidebar → Drawer از چپ؛ Stats Panel → نوار جمع‌وجور inline (نه Drawer، چون فضای کافی نیست)؛ گرید سؤال ۵ ستونه |
| Tablet (`sm`/`md`) | `640–1024px` | گرید سؤال ۱۰–۱۳ ستونه؛ Drawer عرض `360px` |
| Desktop (`lg+`) | `≥ 1024px` | Sidebar ثابت `280px`؛ گرید سؤال ۲۵ ستونه (۲ ردیف)؛ Drawer عرض `420px` |

## نکته‌ی مهم برای هر برنامه‌نویس/جلسه‌ی بعدی

این سند **بعد از** پیاده‌سازی نوشته شده (مستندسازی معکوس)، نه قبل از اون. یعنی اگه تو کد چیزی عوض بشه، این فایل هم باید به‌روزرسانی بشه — در غیر این صورت خودش می‌شه منبع اطلاعات غلط. فایل‌های منبع اصلی برای صحت همیشه اینان: `app/globals.css` (توکن‌ها) و کامپوننت‌های `components/certifications/practice/`.
